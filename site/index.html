Speelka Agent

Universal LLM agent based on the Model Context Protocol (MCP)

Universal LLM agent based on the Model Context Protocol (MCP), with the ability to utilize tools from other MCP servers.

Get Started View on GitHub

Key Features

Context Size Optimization

Reduce client-side context size for more efficient processing and lower token consumption.

Flexible LLM Configuration

Use different LLM providers and settings between client-side and agent for optimized cost and performance.

Precise Agent Definition

Precisely describe agent functionality in prompts for better control and more predictable behavior.

Centralized Tool Management

Manage and orchestrate all tools from a central location for better consistency and control.

Flexible Integration Options

Integrate via MCP stdio, MCP HTTP*, or Simple HTTP API* depending on your system requirements. (*planned)

Built-in Retry Mechanism

Ensure reliability with automatic retries for transient failures and network issues.

Extensible Architecture

Extend and modify system behavior without requiring changes to client-side implementations.

    classDef default fill:#131525,stroke:#8a5cf6,stroke-width:2px,color:#f9fafb;
  1. User request is received by the Agent
  2. Agent processes the request and initializes a Chat session
  3. LLM Service is called with the formatted prompt and available tools
  4. LLM responds with text and/or tool calls
  5. For each tool call, MCP Connector executes the tool on an appropriate server
  6. Tool results are added to the Chat history
  7. The process repeats until LLM issues an "answer" command
  8. Final response is returned to the user through Agent

Usage

Start using Speelka Agent with these configuration examples

Configuration with YAML File

The recommended way to configure Speelka Agent is using a YAML file:

# Save as config.yaml
agent:
  name: "simple-speelka-agent"
  version: "1.0.0"

  # Tool configuration
  tool:
    name: "process"
    description: "Process tool for handling user queries with LLM"
    argument_name: "input"
    argument_description: "The user query to process"

  # LLM configuration
  llm:
    provider: "openai"
    api_key: ""  # For security, set via SPL_LLM_API_KEY environment variable instead
    model: "gpt-4o"
    temperature: 0.7
    prompt_template: "You are a helpful AI assistant. Respond to the following request: {{input}}. Provide a detailed and helpful response. Available tools: {{tools}}"

  # MCP Server connections
  connections:
    mcpServers:
      time:
        command: "docker"
        args: ["run", "-i", "--rm", "mcp/time"]

      filesystem:
        command: "mcp-filesystem-server"
        args: ["/path/to/directory"]

# Runtime configuration
runtime:
  log:
    level: "info"

  transports:
    stdio:
      enabled: true

Important Security Note

Always set sensitive information like API keys via environment variables instead of storing them in configuration files:

export SPL_LLM_API_KEY=your_api_key_here
./speelka-agent --config config.yaml

Environment variables take precedence over file configuration, allowing you to securely manage credentials.

MCP stdio Integration

Add speelka-agent to your MCP client:

# MCP client configuration (YAML format)
mcpServers:
  speelka-agent:
    command: "speelka-agent"
    args: ["--config", "/path/to/agent-config.yaml"]
    environment:
      SPL_LLM_API_KEY: "${OPENAI_API_KEY}"  # Reference environment variable

Docker Integration

Add speelka-agent to your MCP client with Docker:

# MCP client configuration (YAML format)
mcpServers:
  speelka-agent:
    command: "docker"
    args:
      - "run"
      - "-i"
      - "--rm"
      - "-v"
      - "${PWD}/config.yaml:/config.yaml"
      - "-e"
      - "SPL_LLM_API_KEY=${OPENAI_API_KEY}"
      - "ghcr.io/korchasa/speelka-agent:latest"
      - "--config"
      - "/config.yaml"

Additional Examples

More configuration examples to help you integrate Speelka Agent with various tools:

1. Basic Playwright Integration

Set up Speelka Agent with Playwright for browser automation:

# Agent configuration (YAML format)
agent:
  name: "browser-automation-agent"
  tool:
    name: "web-navigation"
    description: "Tool for web browsing and interaction"
    argument_name: "input"
    argument_description: "The user query to process"

  llm:
    provider: "openai"
    api_key: ""  # Set via environment variable
    model: "gpt-4o"
    prompt_template: "You are a web browsing assistant. Help the user navigate websites and perform tasks.\n\nUser request: {{input}}\n\nAvailable tools:\n{{tools}}"

  connections:
    mcpServers:
      playwright:
        command: "npx"
        args: ["@playwright/mcp@latest"]

2. CI/CD Agent Integration

Configure a specialized CI/CD pipeline management agent:

# Agent configuration (YAML format)
agent:
  name: "ci-cd-agent"
  tool:
    name: "change-ci-cd"
    description: "Tool for managing CI/CD pipelines"
    argument_name: "query"
    argument_description: "CI/CD related issue or improvement query"

  llm:
    provider: "openai"
    api_key: ""  # Set via environment variable
    model: "gpt-4o"
    temperature: 0.1
    prompt_template: |
      
      YOU ARE A WORLD-CLASS CI/CD PIPELINES SPECIALIST, RECOGNIZED FOR DESIGNING, MAINTAINING, AND OPTIMIZING HIGHLY RELIABLE, SECURE, AND SCALABLE CONTINUOUS INTEGRATION AND CONTINUOUS DELIVERY PIPELINES.

      ### USER REQUEST ###
      THE USER REQUEST IS AS FOLLOWS:
      **{{query}}**

      ### OBJECTIVE ###
      YOU MUST DESIGN, IMPLEMENT, MONITOR, AND CONTINUOUSLY IMPROVE THE CI/CD PIPELINES THAT AUTOMATE CODE BUILDING, TESTING, SECURITY SCANNING, DEPLOYMENT, AND ROLLBACK OPERATIONS.

      ### AVAILABLE TOOLS ###
      {{tools}}
      

  connections:
    mcpServers:
      filesystem:
        command: "mcp-filesystem-server"
        args: ["./project-root"]

3. Multi-Tool Configuration

Combine Playwright and CI/CD capabilities in a comprehensive development environment:

# Agent configuration (YAML format)
agent:
  name: "development-assistant"
  tool:
    name: "dev-tools"
    description: "Development assistant with browser automation and CI/CD capabilities"
    argument_name: "input"
    argument_description: "The development task to perform"

  llm:
    provider: "openai"
    api_key: ""  # Set via environment variable
    model: "gpt-4o"
    prompt_template: |
      You are a full-stack developer assistant with expertise in both web automation and CI/CD pipelines.

      User request: {{input}}

      Use the appropriate tools to help the user with their development and automation needs.

      Available tools:
      {{tools}}

  connections:
    mcpServers:
      playwright:
        command: "npx"
        args: ["@playwright/mcp@latest"]

      ci-cd-tools:
        command: "speelka-agent"
        args: ["--mode=ci-cd"]

      filesystem:
        command: "mcp-filesystem-server"
        args: ["./project-root"]

4. Minimalist Configuration

A simple, lightweight configuration for quick testing or development:

        args: ["."]

  log:
    level: "info"

Simple Assistant

A basic configuration for a general-purpose assistant:

agent:
  name: simple-speelka-agent
  version: 1.0.0
  tool:
    name: process
    description: Process tool for handling user queries with LLM
    argument_name: input
    argument_description: The user query to process
  llm:
    provider: openai
    api_key: ""  # Set via environment variable for security
    model: gpt-4o
    temperature: 0.7
    prompt_template: >
      You are a helpful AI assistant. Respond to the following request: {{input}}.
      Provide a detailed and helpful response. Available tools: {{tools}}
  connections:
    mcpServers:
      time:
        command: docker
        args:
          - run
          - -i
          - --rm
          - mcp/time
        environment: {}
      filesystem:
        command: mcp-filesystem-server
        args:
          - /path/to/directory
        environment: {}
SPL_AGENT_NAME=simple-speelka-agent
SPL_TOOL_NAME=process
SPL_TOOL_DESCRIPTION=Process tool for handling user queries with LLM
SPL_TOOL_ARGUMENT_NAME=input
SPL_TOOL_ARGUMENT_DESCRIPTION=The user query to process

SPL_LLM_PROVIDER=openai
SPL_LLM_API_KEY=  # Provide your API key here
SPL_LLM_MODEL=gpt-4o
SPL_LLM_TEMPERATURE=0.7
SPL_LLM_PROMPT_TEMPLATE=You are a helpful AI assistant. Respond to the following request: {{input}}. Provide a detailed and helpful response. Available tools: {{tools}}

SPL_MCPS_0_ID=time
SPL_MCPS_0_COMMAND=docker
SPL_MCPS_0_ARGS=run -i --rm mcp/time

SPL_MCPS_1_ID=filesystem
SPL_MCPS_1_COMMAND=mcp-filesystem-server
SPL_MCPS_1_ARGS=/path/to/directory

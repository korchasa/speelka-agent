Speelka Agent

Universal LLM agent based on the Model Context Protocol (MCP)

Universal LLM agent based on the Model Context Protocol (MCP), with the ability to utilize tools from other MCP servers.

Get Started View on GitHub

Key Features

Context Size Optimization

Reduce client-side context size for more efficient processing and lower token consumption.

Flexible LLM Configuration

Use different LLM providers and settings between client-side and agent for optimized cost and performance.

Precise Agent Definition

Precisely describe agent functionality in prompts for better control and more predictable behavior.

Centralized Tool Management

Manage and orchestrate all tools from a central location for better consistency and control.

Flexible Integration Options

Integrate via MCP stdio, MCP HTTP*, or Simple HTTP API* depending on your system requirements. (*planned)

Built-in Retry Mechanism

Ensure reliability with automatic retries for transient failures and network issues.

Extensible Architecture

Extend and modify system behavior without requiring changes to client-side implementations.

    User["Any MCP Client"] --> |"1. Request"| Agent["Speelka Agent"]
    Agent --> |"2. Format prompt"| LLM["LLM Service"]
    LLM --> |"3. Tool calls"| Agent
    Agent --> |"4. Execute tools"| Tools["External MCP Tools"]
    Tools --> |"5. Return results"| Agent
    Agent --> |"6. Process repeat"| LLM

    classDef default fill:#131525,stroke:#8a5cf6,stroke-width:2px,color:#f9fafb;
  1. User request is received by the Agent
  2. Agent processes the request and initializes a Chat session
  3. LLM Service is called with the formatted prompt and available tools
  4. LLM responds with text and/or tool calls
  5. For each tool call, MCP Connector executes the tool on an appropriate server
  6. Tool results are added to the Chat history
  7. The process repeats until LLM issues an "answer" command
  8. Final response is returned to the user through Agent

Usage

Start using Speelka Agent with these example configurations

MCP stdio

Add speelka-agent binary to your MCP client:

{
    "mcpServers": {
        "speelka-agent": {
            "command": "speelka-agent",
            "args": [],
            "environment": {
                "SPL_AGENT_NAME": "simple-speelka-agent",
                "SPL_TOOL_NAME": "process",
                "SPL_TOOL_DESCRIPTION": "Process tool for handling user queries with LLM",
                "SPL_LLM_PROVIDER": "openai",
                "SPL_LLM_API_KEY": "YOUR_API_KEY_HERE",
                "SPL_LLM_MODEL": "gpt-4o",
                "SPL_LLM_PROMPT_TEMPLATE": "You are a helpful AI assistant. Respond to the following request:\\n\\n{{input}}\\n\\nProvide a detailed and helpful response.\\n\\nAvailable tools:\\n{{tools}}",
                "SPL_MCPS_0_ID": "time",
                "SPL_MCPS_0_COMMAND": "docker",
                "SPL_MCPS_0_ARGS": "run -i --rm mcp/time",
                "SPL_MCPS_1_ID": "mcp-filesystem-server",
                "SPL_MCPS_1_COMMAND": "mcp-filesystem-server",
                "SPL_MCPS_1_ARGS": "/Users/korchasa/www/speelka/speelka-agent"
            }
        }
    }
}

Docker Integration

Add speelka-agent to your MCP client with Docker:

{
    "mcpServers": {
        "speelka-agent": {
            "command": "docker",
            "args": ["run", "-i", "--rm",
                "-e", "SPL_AGENT_NAME=simple-speelka-agent",
                "-e", "SPL_TOOL_NAME=process",
                "-e", "SPL_TOOL_DESCRIPTION=Process tool for handling user queries with LLM",
                "-e", "SPL_LLM_PROVIDER=openai",
                "-e", "SPL_LLM_API_KEY=$SPL_LLM_API_KEY",
                "-e", "SPL_LLM_MODEL=gpt-4o",
                "-e", "SPL_LLM_PROMPT_TEMPLATE=You are a helpful AI assistant. Respond to the following request:\\n\\n{{input}}\\n\\nProvide a detailed and helpful response.\\n\\nAvailable tools:\\n{{tools}}",
                "ghcr.io/korchasa/speelka-agent:latest",
                "-e", "SPL_MCPS_0_ID=time",
                "-e", "SPL_MCPS_0_COMMAND=docker",
                "-e", "SPL_MCPS_0_ARGS=run -i --rm mcp/time",
                "-e", "SPL_MCPS_1_ID=mcp-filesystem-server",
                "-e", "SPL_MCPS_1_COMMAND=mcp-filesystem-server",
                "-e", "SPL_MCPS_1_ARGS=/Users/korchasa/www/speelka/speelka-agent"
            ],
            "environment": {
                "SPL_LLM_API_KEY": "YOUR_API_KEY_HERE"
            }
        }
    }
}

Additional Examples

More configuration examples to help you integrate Speelka Agent with various tools:

1. Basic Playwright Integration

Set up Speelka Agent with Playwright for browser automation:

{
    "mcpServers": {
        "playwright": {
            "command": "npx",
            "args": [
                "@playwright/mcp@latest"
            ]
        },
        "speelka-agent": {
            "command": "speelka-agent",
            "args": [],
            "environment": {
                "SPL_AGENT_NAME": "browser-automation-agent",
                "SPL_TOOL_NAME": "web-navigation",
                "SPL_TOOL_DESCRIPTION": "Tool for web browsing and interaction",
                "SPL_LLM_PROVIDER": "openai",
                "SPL_LLM_API_KEY": "YOUR_API_KEY_HERE",
                "SPL_LLM_MODEL": "gpt-4o",
                "SPL_LLM_PROMPT_TEMPLATE": "You are a web browsing assistant. Help the user navigate websites and perform tasks.\\n\\nUser request: {{input}}\\n\\nAvailable tools:\\n{{tools}}",
                "SPL_MCPS_0_ID": "playwright",
                "SPL_MCPS_0_COMMAND": "npx",
                "SPL_MCPS_0_ARGS": "@playwright/mcp@latest"
            }
        }
    }
}

2. CI/CD Agent Integration

Configure a specialized CI/CD pipeline management agent:

{
    "mcpServers": {
        "speelka-agent-for-CI-CD": {
            "command": "speelka-agent",
            "args": [],
            "env": {
                "SPL_AGENT_NAME": "ci/cd agent",
                "SPL_TOOL_NAME": "change-ci-cd",
                "SPL_TOOL_DESCRIPTION": "Tool for managing CI/CD pipelines",
                "SPL_TOOL_ARGUMENT_NAME": "query",
                "SPL_TOOL_ARGUMENT_DESCRIPTION": "CI/CD related issue or improvement query",
                "SPL_LLM_PROVIDER": "openai",
                "SPL_LLM_API_KEY": "YOUR_API_KEY_HERE",
                "SPL_LLM_MODEL": "gpt-4o",
                "SPL_LLM_PROMPT_TEMPLATE": "\nYOU ARE A WORLD-CLASS CI/CD PIPELINES SPECIALIST, RECOGNIZED FOR DESIGNING, MAINTAINING, AND OPTIMIZING HIGHLY RELIABLE, SECURE, AND SCALABLE CONTINUOUS INTEGRATION AND CONTINUOUS DELIVERY PIPELINES.\n\n### USER REQUEST ###\nTHE USER REQUEST IS AS FOLLOWS:  \n**{{query}}**\n\n### OBJECTIVE ###\nYOU MUST DESIGN, IMPLEMENT, MONITOR, AND CONTINUOUSLY IMPROVE THE CI/CD PIPELINES THAT AUTOMATE CODE BUILDING, TESTING, SECURITY SCANNING, DEPLOYMENT, AND ROLLBACK OPERATIONS.\n\n### AVAILABLE TOOLS ###\n{{tools}}\n",
                "SPL_MCPS_0_ID": "mcp-filesystem-server",
                "SPL_MCPS_0_COMMAND": "mcp-filesystem-server",
                "SPL_MCPS_0_ARGS": "./project-root"
            }
        }
    }
}

3. Multi-Tool Configuration

Combine Playwright and CI/CD capabilities in a comprehensive development environment:

{
    "mcpServers": {
        "playwright": {
            "command": "npx",
            "args": [
                "@playwright/mcp@latest"
            ]
        },
        "speelka-agent-dev": {
            "command": "speelka-agent",
            "args": [],
            "environment": {
                "SPL_AGENT_NAME": "development-assistant",
                "SPL_TOOL_NAME": "dev-tools",
                "SPL_TOOL_DESCRIPTION": "Development assistant with browser automation and CI/CD capabilities",
                "SPL_LLM_PROVIDER": "openai",
                "SPL_LLM_API_KEY": "YOUR_API_KEY_HERE",
                "SPL_LLM_MODEL": "gpt-4o",
                "SPL_LLM_PROMPT_TEMPLATE": "You are a full-stack developer assistant with expertise in both web automation and CI/CD pipelines.\\n\\nUser request: {{input}}\\n\\nUse the appropriate tools to help the user with their development and automation needs.\\n\\nAvailable tools:\\n{{tools}}",
                "SPL_MCPS_0_ID": "playwright",
                "SPL_MCPS_0_COMMAND": "npx",
                "SPL_MCPS_0_ARGS": "@playwright/mcp@latest",
                "SPL_MCPS_1_ID": "ci-cd-tools",
                "SPL_MCPS_1_COMMAND": "speelka-agent",
                "SPL_MCPS_1_ARGS": "--mode=ci-cd",
                "SPL_MCPS_2_ID": "mcp-filesystem-server",
                "SPL_MCPS_2_COMMAND": "mcp-filesystem-server",
                "SPL_MCPS_2_ARGS": "./project-root",
                "SPL_RUNTIME_STDIO_ENABLED": "true"
            }
        }
    }
}
